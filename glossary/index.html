wOBA -- Weighted On-base Average
(w1B*1B + w2B*2B + w3B*3B + wHR*HR + wEBB*EBB)/PA
The weights (listed here) are trained on game logs from 2022/23 to present.
Leaders need 3.1 PA per team game played.
aOBP -- Adjusted On-base Percentage
(w1B*1B + w2B*2B + w3B*3B + wHR*HR + wEBB*EBB)/PA
aOBP/aRC/aRC+ are comparable to wOBA/wRC/wRC+, respectively, but satisfy a different objective function: whereas wRC is trained to estimate runs scored from team stat totals, aOBP is trained to estimate On-base Percentage from inferred individual base target volume. A core assumption of conventional linear weights methodology is that 'things' happen more-or-less randomly. In dartball, this is demonstrably false: bases are chosen, e.g. triples are typically only attempted with runners on base, and their success/failure has a disproportionate impact on game outcomes. This causes several problems when using wOBA to assess player skill: 1) wRC over-values triples and errors relative to singles. 2) wRC under-penalizes outs when attempting triples (this could be resolved by separating single and triple outs in the stat-sheet, but multiple 'out states' creates ambiguity in defining wOBA). Consequently, a player's wOBA is heavily influenced by his role/position in the batting order, i.e. the same player will tend to have a higher wOBA when slugging vs. batting lead-off. 3) The computed wRC/wOBA weights are heavily influenced by team strategy (intentionally loading bases or attempting doubles will result in very different calculated weights than a 'traditional' strategy). aOBP circumvents all of these issues by ignoring runs entirely. It is instead optimized as a target-neutral skill estimate, i.e. a player's aOBP should tend toward the same value, regardless of the base attempted, and his expected single or triple OBP can be estimated by dividing his aOBP by the respective base weight. The weights (listed here) are trained on game logs from 2022/23 to present.
Leaders need 3.1 PA per team game played.
OPS+ -- OPS+
100*[OBP/lg OBP + SLG/lg SLG - 1]
Leaders need 3.1 PA per team game played.
wRC+ -- Weighted Runs Created Plus
100*[(wRC/PA)/(lg wRC/PA)]
Leaders need 3.1 PA per team game played.
aRC+ -- Adjusted Runs Created Plus
100*[(aRC/PA)/(lg aRC/PA)]
See notes under aOBP.
Leaders need 3.1 PA per team game played.
wRC -- Weighted Runs Created
[(wOBA - lg wOBA)/(wOBA scale) + (lg wRC/PA)]*PA
aRC -- Adjusted Runs Created
[(aOBA - lg aOBA)/(aOBA scale) + (lg aRC/PA)]*PA
See notes under aOBP. aRC is to aOBP as wRC is to wOBA. After aOBP weights are calculated, the parameters needed to calculate aRC are solved for by regressing aOBP against runs scored. While wRC is slightly better than aRC at estimating game runs scored, aRC is better at assigning credit for those runs to players, and less biased.
gaRC -- Game-standardized Adjusted Runs Created.
(aRC/PA)*(lg PA/G)*G
A counting statistic that allows for fairer comparison between players with discrepant PA/G. In principle, it should eliminate all factors outside of player control.
Luck -- Luck
Difference between team runs scored and team wRC.
